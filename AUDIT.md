McOS 1.0 — результаты аудита и ремонта
======================================

Объём проверки
--------------
Проверены загрузчик, установщик, файловая система, запись состояния, корзина,
резервные копии и восстановление, McNet, Redstone Center, периферия, touch-режим,
приложения, McStore, обработка событий и миграция McOS 1.x/2.0.

Ниже перечислены ошибки и недочёты, найденные статическим аудитом и тестами.
Фраза «исправлено» означает, что соответствующее изменение присутствует в этой
сборке. Это не является обещанием отсутствия вообще всех возможных ошибок внутри
Minecraft или сторонних модовых peripherals.

Критические исправления
-----------------------
1. Установщик мог удалить старый startup.lua, не записать новый и оставить систему
   без загрузчика. Исправлено: изменяемый путь заранее включается в rollback;
   протестирован отказ именно после удаления startup.lua.
2. Запись config/state могла обнулить файл при ошибке диска или move. Исправлено:
   временный файл, чтение-проверка, сохранение предыдущей версии и rollback.
3. После прерывания атомарной записи предыдущая версия могла остаться в sidecar,
   а McOS загружала пустые настройки. Исправлено раннее восстановление до чтения БД.
4. Restore мог завершиться между заменой каталогов и оставить смесь старых и новых
   данных. Исправлено полное staging-восстановление с restore journal и откатом при
   следующей загрузке после потери питания.
5. Ошибка на позднем шаге Restore не откатывала уже заменённые ранние каталоги.
   Исправлен транзакционный rollback всей последовательности.
6. Резервная копия не включала содержимое корзины. Теперь сохраняются state-файлы,
   /mcos/user, /mcos/apps и /mcos/trash.
7. Recovery показывал каталоги установочных backup без recovery manifest.
   Теперь отображаются только реальные восстанавливаемые копии.

Файловая система и корзина
--------------------------
8. Поиск выполнялся по всему диску вместо выбранной папки. Исправлен scoped recursive
   search с ограничениями глубины и количества результатов.
9. Можно было скопировать папку внутрь самой себя. Добавлена проверка границ пути.
10. Имена «.», «..» и пути с разделителями позволяли path traversal. Добавлена
    канонизация и безопасные имена.
11. McStore/передача файлов допускали опасные или служебные имена. Зарезервированы
    суффиксы транзакций и очищаются управляющие символы.
12. Системные БД, ядро, startup.lua, backup и внутренности корзины можно было
    переименовать/удалить из Files. Добавлена модель protected targets/containers.
13. Корень /mcos, /mcos/apps и /mcos/user можно было удалить как обычную папку.
    Структурные корни теперь защищены, их обычное содержимое остаётся управляемым.
14. Повторное удаление файла внутри корзины могло создать рекурсивную корзину.
    Исправлено.
15. Восстановление вложенного файла принимало его за отдельный top-level trash item
    и портило индекс родительской папки. Исправлено.
16. Устаревшие записи trash.db не очищались. Теперь индекс сверяется с диском.
17. При ошибке сохранения trash.db файл уже перемещался и становился «сиротой».
    Индекс теперь фиксируется до удаления, а restore выполняет rollback при отказе.
18. Создание файлов, заметок и приложений могло молча затереть существующее.
    Добавлены проверки и подтверждения.
19. Переименование могло заменить существующий объект. Добавлена проверка target.
20. File Manager мог остаться в удалённой/отключённой папке. Теперь возвращается в /.
21. Просмотрщик пытался загрузить файл любого размера. Добавлен лимит 512 KiB.
22. Paint мог открыть системный файл или каталог. Добавлена проверка.

McNet
-----
23. Ping запускал sleep(), который съедал PONG. Переведено на request ID и общий
    event loop с timeout.
24. rednet.send воспринимался как подтверждение доставки. Теперь сообщение и файл
    считаются доставленными только после ACK принимающей McOS.
25. Получатель подтверждал сообщение до сохранения истории. ACK теперь отправляется
    только после успешной записи mcnet.db.
26. Переписка пропадала после перезагрузки. Inbox сохраняется и ограничен 100 записями.
27. Одинаковые имена полученных файлов перезаписывались. Используются уникальные пути.
28. Входящий файл не проверял тип, размер и свободное место. Добавлены проверки,
    лимит 128 KiB и безопасная обработка ошибки getFreeSpace.
29. Remote Redstone и Remote Run управлялись одним переключателем. Разделены.
30. Remote-команды принимались от любого ID. Требуется trusted peer и разрешение.
31. Ошибка Lua-конструкции trusted and nil or true не позволяла снять trust.
    Заменено явным if/else.
32. Remote Run не проверял путь, размер очереди и аргументы. Добавлена валидация.
33. Remote Run не сообщал итог. Добавлены queued ACK и отдельный result ACK.
34. Remote-задачи могли ждать до следующего нажатия пользователя. Сервисный timer
    будит desktop каждые 0,5 секунды.
35. Текстовый ввод через read() блокировал McNet и таймеры. Реализован собственный
    event-driven редактор строки с char/paste/backspace/Enter/Esc.
36. Список peers бесконечно сохранял исчезнувшие устройства. Добавлен TTL 120 секунд.
37. Данные peer/message могли раздувать интерфейс. Поля обрезаются и проверяются.

Redstone и таймеры
------------------
38. Сохранённые таймеры/правила могли содержать неверные стороны, значения и типы и
    уронить service loop. Данные валидируются и ограничиваются.
39. Аналоговые значения не везде ограничивались диапазоном 0–15. Добавлен clamp.
40. Pulse не возвращал предыдущий уровень. Теперь сохраняет старый output.
41. Нулевые/отрицательные/гигантские задержки принимались без проверки. Исправлено.
42. Повреждённые scenes могли уронить сортировку или применение. Добавлена очистка.
43. Clock и системные таймеры обновлялись только после ввода. Добавлен service timer.
44. Автоматизация останавливалась во время boot animation и теста монитора/звука.
    Используются cooperative delays.

Интерфейс, monitor и peripherals
--------------------------------
45. monitor_touch обрабатывался нецентрализованно и не во всех приложениях.
    Активный monitor_touch нормализуется в mouse_click.
46. Отключение активного монитора оставляло term.redirect на мёртвом объекте.
    Добавлен возврат на компьютер и term_resize.
47. Слишком крупный scale мог сделать интерфейс непригодным. Проверяется минимум
    32×16; неудачное изменение scale откатывается.
48. Turtle touch-layout пересекался с нижней панелью на экране высотой 16 строк.
    Элементы перестроены в компактную сетку.
49. Вызовы getNames/getType/getMethods/wrap могли падать при отключении устройства.
    Основные пути обёрнуты pcall и проверяют результат.
50. Notes показывал каталоги как заметки. Теперь только файлы.
51. Inventory Viewer не сортировал slots и не обрабатывал disconnect. Исправлено.
52. System Monitor не обновлялся автоматически и мог упасть на getFreeSpace.
    Добавлен refresh и безопасный fallback.
53. Ctrl+T мог уничтожить весь UI из обычного диалога. В модальных окнах он
    преобразуется в Escape; raw-сценарии загрузки/lock всё ещё видят terminate.
54. Длинные CraftOS tools блокировали систему. На Advanced Computer они открываются
    через shell.openTab/multishell, когда API доступен.

Приложения и McStore
--------------------
55. Calculator допускал произвольный Lua-код. Оставлен только whitelist арифметики и
    пустое окружение load().
56. Notes могла очистить существующую заметку. Теперь предлагает открыть без очистки.
57. Music Player пытался воспроизводить DFPWM без decoder и не ждал audio buffer.
    Используется cc.audio.dfpwm и speaker_audio_empty.
58. Printer Center не проверял newPage/endPage и не переносил длинные строки.
    Добавлена пагинация 25×21 и обработка ошибок.
59. McStore принимал не-HTTPS URL и опасные имена. Исправлено.
60. McStore сначала загружал ответ целиком и только потом проверял 256 KiB.
    Теперь потоковое чтение останавливается на лимите.
61. McStore мог записать синтаксически повреждённый Lua. Добавлен load-check.
62. URL с токеном/query попадал в system.log. Query и fragment удаляются из лога.
63. Ошибка внешнего приложения могла уронить desktop. Запуск изолирован pcall,
    ошибка записывается в log и Notification Center.

Установщик и миграция
---------------------
64. Payload не компилировался до изменения файлов. Теперь все Lua-файлы проверяются.
65. startup.lua мог заменяться раньше ядра. Теперь он устанавливается последним.
66. Записи установщика не проверялись чтением. Добавлена верификация temporary file.
67. При частичном отказе не было полного отката заменённых файлов. Исправлено.
68. Старый пользовательский каталог и внешние apps могли быть перезаписаны.
    Установщик сохраняет /mcos/user, настройки и уже существующий hello app.
69. Совместимые параметры McOS 1.x не мигрировали. Добавлен безопасный импорт.
70. Старый общий allowRemote мог молча включить опасные возможности. При миграции
    обе новые remote permissions принудительно выключаются.

Оставшиеся ограничения, которые нельзя честно назвать «исправленными»
--------------------------------------------------------------------
1. Сборка протестирована в Lua 5.2/макете CC:Tweaked, но не запущена в реальном мире
   Minecraft. Сторонний peripheral может иметь несовместимую реализацию.
2. Rednet по своей природе не шифрует и не аутентифицирует отправителя. Trusted ID
   снижает риск случайной команды, но не защищает от намеренного spoofing.
3. PIN — блокировка интерфейса, а не шифрование диска.
4. McStore apps не sandboxed и получают обычные права компьютера.
5. Без multishell внешняя длительная программа временно блокирует фоновые службы.
6. Аппаратное выключение ровно во время замены startup.lua установщиком остаётся
   фундаментальным риском файловой системы без атомарного replace; предыдущая версия
   сохраняется в timestamped backup, а обычные программные ошибки откатываются.
